# game_little_light

《微孔：疫苗坠落》是一个末世题材剧情推理卡牌生存游戏项目。仓库包含两部分：

- 主游戏 MVP：Vite + React 实现的可交互剧情卡牌游戏。
- 游戏介绍 Demo：位于 `game_intro_demo/` 的静态介绍页。

玩家带领沈砚小队进入废弃微孔过滤工厂，寻找坠毁运输机上的疫苗原液的故事。

本仓库中的小说文本、沈砚线剧本整理文件、素材需求清单、游戏美术素材和 Demo 素材均由 Codex 辅助生成，并整理进当前项目结构中。

- `沈砚线_素材需求清单_v0.2.md`：围绕沈砚线整理的素材需求。

## 游戏概览

《微孔：疫苗坠落》的核心玩法是剧情选择 + 线索收集 + 数值状态驱动的分支结局。

玩家在每个事件节点阅读剧情、查看人物与线索，并做出行动选择。选择会改变队伍状态、暴露风险、冷链完整度、疫苗持有人、沈砚感染状态等变量；这些变量会影响后续可选路线，并最终导向不同结局。

核心系统包括：

- 剧情事件卡：推进探索、遭遇、撤离与终局选择。
- 线索卡：记录灰隼信号、运输机尾号、破碎疫苗、音爆弹、冷舱等关键信息。
- 人物档案：展示主要角色身份、立场和剧情功能。
- 状态数值：真相、样本、队伍、暴露、冷链、林禾判断、沈砚感染。
- 分支变量：阿拓是否救过许燃、是否取得音爆弹、疫苗交给谁、沈砚是否感染或冷舱休眠。

## 人物与故事线

当前可玩版本包含 1 条主角故事线：沈砚线。

沈砚线围绕“疫苗是否仍值得护送”展开。玩家需要在任务命令、现场线索、队友信任和撤离代价之间做判断。

主要人物共 8 位：

- 沈砚：安全区行动队长，玩家扮演角色。
- 林禾：研究员，负责判断疫苗与感染异常。
- 阿拓：维修与救援角色，影响冷舱和许燃相关分支。
- 许燃：通讯员，后续暴露为白鸮。
- 白鸮：许燃的隐藏身份，对抗局潜伏者。
- 灰隼：留下“疫苗无效”信号的对抗局特工。
- 邵戟：对抗局行动人物，推动终局对峙。
- 巢声者：感染者巢穴相关角色。

## 结局

当前版本共有 8 个可达结局：

- E01：异化的希望
- E02：最后一枪
- E03：黑潮之后
- E04：冷舱里的队长
- E05：带回问题的人
- E06：两枚铭牌
- E07：失去疫苗，留下沈砚
- E08：活着的失败者

主目录下的 `storylines.md` 是当前游戏故事线总览文档，包含沈砚线主线、三大分歧、关键变量、8 个结局路径和 Mermaid 图解。

## 当前开发进度

当前项目处于可玩 MVP 阶段，重点完成了沈砚线的剧情闭环、分支结构和基础交互体验。

已完成内容：

- 主游戏框架：基于 Vite + React + TypeScript 搭建，可本地运行与生产构建。
- 沈砚线 MVP：已实现从序章到终局的完整可玩流程。
- 分支与结局：已实现 1 条主角故事线、3 个主要分歧、8 个可达结局。
- 选择与变量系统：已接入疫苗持有人、阿拓是否救过许燃、音爆弹、沈砚感染状态、疫苗归属等关键变量。
- 线索与人物面板：已实现线索卡、人物档案、行动记录和状态数值展示。
- 视觉素材：已整理背景、人物、线索、结局图等素材，并接入当前游戏界面。
- 游戏介绍 Demo：已完成 `game_intro_demo/` 静态介绍页，并在构建时复制到 `dist/game_intro_demo/`。
- 文档整理：已补充素材索引、沈砚线脚本、素材需求清单、故事线图解文档。
- 路线校验：已提供 `npm run validate:routes`，用于检查剧情节点与 8 个结局是否可达。

当前仍属于原型版本，主要目标是验证剧情结构、交互节奏和结局分支，不代表最终商业化完成度。

## 后续推进方向

下一阶段建议围绕“内容扩展、系统打磨、视觉演出、发布体验”推进：

- 剧情内容扩展：继续补充或拆分更多主角线，例如林禾线、阿拓线、许燃/白鸮线，让不同角色拥有独立视角和交叉结局。
- 章节演出强化：为关键节点增加过场、角色立绘状态、受伤/感染表现、白鸮暴露演出和结局 CG 节奏。
- 玩法系统深化：强化线索卡用途，让线索不仅用于展示，也能解锁选项、影响判断或改变结局条件。
- 数值反馈优化：让真相、样本、队伍、暴露、冷链、判断、感染等状态对玩家更可理解，并在关键选择前后给出清晰反馈。
- 存档与回溯：增加章节选择、结局图鉴、路线回看、关键选择回溯，降低多结局探索成本。
- 素材完善：补充缺失道具图、感染态角色、关键 CG、UI 图标和移动端适配素材。
- 移动端体验：优化触屏交互、侧栏打开方式、字号、按钮热区和小屏幕信息层级。
- 音频与氛围：加入环境音、事件音效、结局音乐和关键选择反馈音。
- 发布部署：补充 GitHub Pages 或其他静态部署配置，让主游戏和 Demo 可直接在线访问。
- 测试与质量：继续扩展路线校验、UI 截图检查和基础交互测试，避免新增剧情后出现断线或不可达结局。

## 运行

```bash
npm install
npm run dev
```

当前开发服务器会显示本地 URL，例如 `http://localhost:5174/`。

开发模式入口：

- 主游戏：`/`
- 游戏介绍 Demo：`/game_intro_demo/`

## 构建

```bash
npm run build
```

构建后会生成：

- `dist/`：主游戏生产构建。
- `dist/game_intro_demo/`：随主游戏一起发布的介绍 Demo。
